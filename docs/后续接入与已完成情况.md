# CampusPilot 已完成情况与后续金蝶接入

## 一、当前已完成

### 1. 官方 KWC 控件

- 已由 `@kdcloudjs/kd-custom-control-cli` 官方模板承载。
- 控件标识为 `campuspilot`，平台入口使用 `KDApi.register('campuspilot', ...)`。
- 元数据目标为 `KWCFormModel`，可直接用于苍穹页面设计器的自定义控件方案。
- 已实现 `pageTitle`、`apiBase`、`defaultRole`、`enableAgent`、`theme` 五项可配置属性。
- 页面已覆盖驾驶舱、学生画像、课程成绩、学习行为、风险预警和 Agent 入口。

### 2. Java 业务后端

- 已提供 `/api/campuspilot/*` 业务接口和角色数据范围控制。
- 已实现预警闭环：风险识别、预警单、辅导员确认、导师帮扶、学生反馈、复评结案。
- 已接入四个金蝶业务对象查询接口：学生画像、课程成绩、学习行为、风险预警。
- 已实现 `KingdeeDataClient`：后端携带 `accessToken` 查询平台，并将 `data.rows` 转换为页面字段。
- 平台未配置、接口超时或不可达时，会自动回退到本地演示数据；接口响应头 `X-CampusPilot-Data-Mode` 可用于判断当前数据来源。

### 3. Agent 集成边界

- KWC 只调用 Java 后端的 `POST /api/campuspilot/agent/chat`。
- Java 后端在配置 `CAMPUSPILOT_AGENT_API_URL` 与 `CAMPUSPILOT_AGENT_API_KEY` 后才请求真实 Agent。
- 调用 Agent 时会将学生画像、课程成绩、学习行为和风险预警作为 `campusContext` 一并发送。
- 未配置真实 Agent 时，系统仍提供本地兜底回答，适合离线展示。

## 二、首次接入金蝶环境

1. 在金蝶环境创建或确认第三方应用，并为业务对象查询 API 分配权限。
2. 取得可用 `accessToken`，仅配置在 Java 后端服务器的环境变量中。
3. 启动 Java 后端，确认 `GET /api/campuspilot/integration-status` 返回正确状态。
4. 在 KWC 项目执行 `npm ci`、`npm run lint`、`npm run build`。
5. 使用 `npx kd-custom-control-cli deploy` 上传控件元数据；若只更新元数据，可使用 `npx kd-custom-control-cli deploy -f`。
6. 在页面设计器中添加自定义控件，方案 ID 使用 `campuspilot`，并将 `apiBase` 配置为 Java 后端根地址。
7. 访问页面，检查浏览器 Network 中 `overview`、`students`、`warnings` 等接口是否返回 `200`。

## 三、手工上传资源包时的注意事项

如果环境采用“控件方案 → 资源文件 ZIP”的手工上传模式，ZIP **根目录**必须直接存在：

```text
index.js
campuspilot.js-meta.kwc
js/
lang/
```

不能额外套一层 `campuspilot/` 或 `code/code/campuspilot/` 目录；否则平台会提示“压缩包根目录没有 index.js 入口文件”。资源来自 KWC 项目的 `dist/` 目录。

## 四、验收建议

```text
1. npm run lint 通过
2. npm run build 成功
3. Java build.ps1 成功
4. /api/campuspilot/health 返回正常
5. /api/campuspilot/integration-status 显示预期数据模式
6. 金蝶页面中控件可加载，且业务接口请求为 200
```

如果平台控制台出现 `kd-libs` 中 React 版本不满足的错误，优先检查平台宿主资源版本与缓存；该错误来自平台 `web-app` 的共享依赖，不能通过向原生 KWC 控件额外打包 React 来修复。
